### 1. Rebuild + redeploy pour forcer `lastmod` à J+7

Le `lastmod` du sitemap est figé au 2026-04-24 (date du build Sprint 5). Google peut interpréter une date vieille de 7 jours comme "pas de changement". Déclencher un build Vercel vide (`git commit --allow-empty`) pour régénérer le sitemap avec `lastmod=2026-05-01`, puis re-pinguer IndexNow.

```bash
git commit --allow-empty -m "chore: rebuild sitemap lastmod J+7"
git push origin main
# puis après deploy:
curl -X POST -H "Authorization: Bearer $INDEXNOW_SECRET" https://www.ghostmeta.online/api/indexnow
```

### 2. Ajouter `dateModified` dynamique dans le prerender des articles

Actuellement `dateModified` est hardcodé à `2026-02-17` dans tous les articles. Google favorise les contenus récents pour le rich result BlogPosting. Passer `dateModified` à la date du dernier build dans `scripts/prerender.mjs` (même logique que `lastmod` du sitemap).

### 3. Si /pricing est toujours "Crawled – not indexed" après J+14

Envisager l'ajout d'un lien interne prominent vers `/pricing` depuis la home (le footer ou le CTA principal). Google indexe en priorité les pages avec des liens internes depuis des pages déjà indexées. Actuellement `/pricing` n'apparaît peut-être pas dans les liens crawlables du bot-content de la home.
