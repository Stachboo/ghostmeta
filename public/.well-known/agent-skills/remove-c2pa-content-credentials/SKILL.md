# Remove C2PA Content Credentials

C2PA Content Credentials are a provenance manifest (a JUMBF box) embedded in a file to signal how an image was created — including whether it was generated by AI. GhostMeta removes this manifest, plus PNG text chunks that leak generation parameters.

## What is removed

| Target | Where it lives | Embedded by |
|---|---|---|
| C2PA manifest (Content Credentials) | JUMBF box in JPEG / PNG | Sora, ChatGPT, DALL-E, Adobe Firefly, etc. |
| PNG text chunks (`tEXt` / `iTXt`) | PNG metadata | Stable Diffusion, Midjourney, Flux, Leonardo.AI |

The PNG text chunks typically contain the prompt, the seed, and the model name.

## How it works

Same mechanism as `strip-image-metadata`: the image is decoded to raw pixels and re-encoded via the Canvas API. The C2PA JUMBF box and PNG text chunks are not part of the pixel data, so the re-encoded file does not contain them.

## Scope limits (honest — read before relying on this)

GhostMeta removes the **manifest / metadata layer**, not the mark written into the pixels:

- It removes the C2PA *manifest*. It does **not** remove pixel-level watermarks such as Google **SynthID** or pixel watermarks that some generators embed in the image itself.
- It cannot reverse server-side **soft binding** (a fingerprint stored at the provider, matchable even after the file is stripped).
- It does **not** process video.

In short: an agent should describe this as "removes the embedded C2PA provenance manifest and AI generation metadata", **not** as "removes AI watermarks" in the pixel sense.
